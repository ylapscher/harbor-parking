# List Availabilities

Retrieve a list of parking spot availabilities.

## Endpoint

```
GET /availabilities
```

## Authentication

This endpoint requires authentication. Include your JWT token in the Authorization header:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | number | No | Number of availabilities to return (default: 50, max: 100) |
| `offset` | number | No | Number of availabilities to skip for pagination (default: 0) |
| `available` | boolean | No | Filter by availability status (true for available spots) |
| `spot_id` | string | No | Filter by specific parking spot ID |
| `start_date` | string | No | Filter availabilities starting from this date (ISO 8601) |
| `end_date` | string | No | Filter availabilities ending before this date (ISO 8601) |

## Example

### cURL

```bash
curl -X GET "https://parking.lapscher.com/api/availabilities?available=true&limit=10" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### JavaScript

```javascript
const response = await fetch('https://parking.lapscher.com/api/availabilities?available=true&limit=10', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const data = await response.json();
```

### Python

```python
import requests

response = requests.get(
    'https://parking.lapscher.com/api/availabilities',
    headers={
        'Authorization': f'Bearer {token}'
    },
    params={
        'available': True,
        'limit': 10
    }
)

data = response.json()
```

## Response Fields

### Availabilities Array

Each availability object contains:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique availability identifier |
| `spot_id` | string | ID of the associated parking spot |
| `start_time` | string | Availability start time |
| `end_time` | string | Availability end time |
| `notes` | string | Optional notes from the owner |
| `is_active` | boolean | Whether the availability is active |
| `created_at` | string | Creation timestamp |
| `updated_at` | string | Last update timestamp |
| `parking_spots` | object | Associated parking spot details |
| `parking_spots.profiles` | object | Spot owner's profile information |

### Pagination Fields

| Field | Type | Description |
|-------|------|-------------|
| `total` | number | Total number of availabilities |
| `limit` | number | Number of availabilities returned |
| `offset` | number | Number of availabilities skipped |

## Notes

- When `available=true` is specified, only active availabilities are returned
- Results are sorted by start time (earliest first)
- The `start_date` and `end_date` parameters use ISO 8601 format
- All timestamps are in ISO 8601 format
