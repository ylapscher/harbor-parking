# Create Availability

Create a new availability window for a parking spot owned by the authenticated user.

## Endpoint

```
POST /availabilities
```

## Authentication

This endpoint requires authentication. Include your JWT token in the Authorization header:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

```json
{
  "spot_id": "123e4567-e89b-12d3-a456-426614174000",
  "start_time": "2024-01-20T09:00:00Z",
  "end_time": "2024-01-20T18:00:00Z",
  "notes": "Available while at work"
}
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `spot_id` | string | Yes | ID of the parking spot |
| `start_time` | string | Yes | Availability start time (ISO 8601) |
| `end_time` | string | Yes | Availability end time (ISO 8601) |
| `notes` | string | No | Optional notes about the availability |

## Response

### Success Response (201 Created)

```json
{
  "availability": {
    "id": "789e0123-e89b-12d3-a456-426614174002",
    "spot_id": "123e4567-e89b-12d3-a456-426614174000",
    "start_time": "2024-01-20T09:00:00Z",
    "end_time": "2024-01-20T18:00:00Z",
    "notes": "Available while at work",
    "is_active": true,
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  }
}
```

### Error Responses

#### 400 Bad Request

```json
{
  "error": "Validation failed",
  "details": {
    "spot_id": {
      "_errors": ["Spot ID is required"]
    },
    "start_time": {
      "_errors": ["Start time must be in the future"]
    },
    "end_time": {
      "_errors": ["End time must be after start time"]
    }
  }
}
```

#### 401 Unauthorized

```json
{
  "error": "Authentication required"
}
```

#### 403 Forbidden

```json
{
  "error": "You can only create availabilities for your own spots"
}
```

#### 404 Not Found

```json
{
  "error": "Parking spot not found"
}
```

## Example

### cURL

```bash
curl -X POST "https://parking.lapscher.com/api/availabilities" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "spot_id": "123e4567-e89b-12d3-a456-426614174000",
    "start_time": "2024-01-20T09:00:00Z",
    "end_time": "2024-01-20T18:00:00Z",
    "notes": "Available while at work"
  }'
```


## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique availability identifier |
| `spot_id` | string | ID of the associated parking spot |
| `start_time` | string | Availability start time |
| `end_time` | string | Availability end time |
| `notes` | string | Optional notes about the availability |
| `is_active` | boolean | Whether the availability is active (initially true) |
| `created_at` | string | Creation timestamp |
| `updated_at` | string | Last update timestamp |

## Notes

- Only the spot owner can create availabilities for their spots
- Start time must be in the future
- End time must be after start time
- New availabilities are created with `is_active: true`
- All timestamps must be in ISO 8601 format
