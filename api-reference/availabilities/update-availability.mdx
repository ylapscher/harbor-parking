# Update Availability

Update details of an availability window owned by the authenticated user.

## Endpoint

```
PUT /availabilities
```

## Authentication

This endpoint requires authentication. Include your JWT token in the Authorization header:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

```json
{
  "id": "789e0123-e89b-12d3-a456-426614174002",
  "start_time": "2024-01-20T10:00:00Z",
  "end_time": "2024-01-20T19:00:00Z",
  "notes": "Available while at work - updated hours",
  "is_active": true
}
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | ID of the availability to update |
| `start_time` | string | No | New start time (ISO 8601) |
| `end_time` | string | No | New end time (ISO 8601) |
| `notes` | string | No | New notes about the availability |
| `is_active` | boolean | No | Whether the availability is active |

## Response

### Success Response (200 OK)

```json
{
  "availability": {
    "id": "789e0123-e89b-12d3-a456-426614174002",
    "spot_id": "123e4567-e89b-12d3-a456-426614174000",
    "start_time": "2024-01-20T10:00:00Z",
    "end_time": "2024-01-20T19:00:00Z",
    "notes": "Available while at work - updated hours",
    "is_active": true,
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T11:45:00Z"
  }
}
```

### Error Responses

#### 400 Bad Request

```json
{
  "error": "Validation failed",
  "details": {
    "start_time": {
      "_errors": ["Start time must be in the future"]
    },
    "end_time": {
      "_errors": ["End time must be after start time"]
    }
  }
}
```

#### 401 Unauthorized

```json
{
  "error": "Authentication required"
}
```

#### 403 Forbidden

```json
{
  "error": "You can only update availabilities for your own spots"
}
```

#### 404 Not Found

```json
{
  "error": "Availability not found"
}
```

## Example

### cURL

```bash
curl -X PUT "https://parking.lapscher.com/api/availabilities" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "789e0123-e89b-12d3-a456-426614174002",
    "end_time": "2024-01-20T19:00:00Z",
    "notes": "Extended hours"
  }'
```

### JavaScript

```javascript
const response = await fetch('https://parking.lapscher.com/api/availabilities', {
  method: 'PUT',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: '789e0123-e89b-12d3-a456-426614174002',
    end_time: '2024-01-20T19:00:00Z',
    notes: 'Extended hours'
  })
});

const data = await response.json();
```

### Python

```python
import requests

response = requests.put(
    'https://parking.lapscher.com/api/availabilities',
    headers={
        'Authorization': f'Bearer {token}',
        'Content-Type': 'application/json'
    },
    json={
        'id': '789e0123-e89b-12d3-a456-426614174002',
        'end_time': '2024-01-20T19:00:00Z',
        'notes': 'Extended hours'
    }
)

data = response.json()
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique availability identifier |
| `spot_id` | string | ID of the associated parking spot |
| `start_time` | string | Availability start time |
| `end_time` | string | Availability end time |
| `notes` | string | Optional notes about the availability |
| `is_active` | boolean | Whether the availability is active |
| `created_at` | string | Creation timestamp |
| `updated_at` | string | Last update timestamp |

## Notes

- Only the spot owner can update availabilities for their spots
- At least one field must be provided in the request body
- Start time must be in the future
- End time must be after start time
- The `updated_at` timestamp is automatically updated
- All timestamps must be in ISO 8601 format
